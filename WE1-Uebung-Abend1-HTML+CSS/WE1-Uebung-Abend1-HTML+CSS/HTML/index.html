Encoding / Umlaute

Sehen Sie sich die folgende Datei an: files/aoeuml.html. Einige Zeichen werden nicht richtig dargestellt. Finden Sie das Problem und beheben Sie es. 

<meta charset="ascii">

Umlaute sind nicht ASCII kompatibel und werden von UTF8 mit mehr als einem Byte abgespeichert. Durch die Interpretation des UTF8 codierten Dokumentes als ASCII werden längere Zeichen als einzelne interpretiert wodurch ein falsches Zeichen dargestellt wird. Lösung: 

<meta charset="utf-8">

Zu beachten: <meta charset="utf-8"> muss im File gesetzt werden. Das Encoding ist beim Einlesen vom File relevant. Eine Änderung im DOM (Browser) bewirkt nichts.

Tesla Model S
=============

The Tesla Model S is a full-sized all-electric five-door, luxury liftback, produced by Tesla Motors,

Debugging - Fehler erkennen

Das gegebene HTML debug.html stellt die Bestätigungsseite zu einem Formular dar. Leider funktioniert diese Seite nicht wie gewünscht. Nutzen Sie die Development Tools ihres Browser um die Fehler zu finden.

Hinweis: Öffnen Sie das Element Tab in den Development Tools und analysieren Sie den vom Parser generierten DOM Baum. Vergleichen Sie den Baum mit dem HTML im File. Die Development Tools lassen sich in den gängigen Browsern unter Windows mit F12 öffnen. OSX/Mac Chrome: [COMMAND+OPT+I]).

  1. Vergleichen Sie die vom Browser interpretierte Struktur mit der im HTML gegebenen Struktur. Welche Mutation hat der Browser vorgenommen?

    Der Browser findet den abschliessenden SCRIPT-Tag nicht; sämtliches HTML wird als JavaScript eingelesen.

    Das Script-Element ist kein Void-Element. Es erlaubt auch keine Tag-Ommission: Es muss sowohl das öffnende, wie das schliessende Tag gesetzt werden. Siehe hierzu MDN Definition des Script Elements W3C Definition des Script Elements

  2. Finden Sie die Fehler und korrigieren Sie diese im HTML-Editor. Der W3C Validator validator.w3.org/#validate_by_upload+with_options kann auch Korrektur-Tips anzeigen (Advanced > [X] Clean up Markup with HTML-Tidy).

    Der Browser findet den abschliessenden SCRIPT-Tag nicht; sämtliches HTML wird als JavaScript eingelesen.

  3. Lesen Sie im HTML Standard die Eigenschaften des SCRIPT Elements HTML5 (siehe auch tag omission) nach. Warum zeigt der Browser eine derartige Reaktion aufs inkorrekte HTML?

    Das SCRIPT-Element erfordert in jedem Fall einen vollen abschliessenden Tag. Tag Omission (d.h. der Tag-End wird automatisch gesetzt), wird vom SCRIPT Element nicht unterstützt. Vorsicht: Ein /> gilt im HTML nicht als abschliessenden Tag; dazu müsste XHTML, welches aber wiederum den vollen End-Tag als REQUIRED voraussetzt. Es gibt weitere HTML-Elemente (wie beispielsweise IFRAME / OPTION / ...), welche ebenfalls einen vollen abschliessenden Tag erfordern.
